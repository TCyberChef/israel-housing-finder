## Directory Purposes

**`src/`:**
- Purpose: React application source code
- Contains: Components, styles, entry points
- Key files: `index.js` (entry), `App.js` (root component)

**`src/config/`:**
- Purpose: External service client configuration
- Contains: Supabase client initialization
- Key files: `supabase.js`

**`public/`:**
- Purpose: Static assets served directly without processing
- Contains: HTML shell, favicon, PWA manifest, robots.txt
- Key files: `index.html` (entry point), `manifest.json` (PWA config)

**`.github/workflows/`:**
- Purpose: GitHub Actions CI/CD automation
- Contains: Deployment workflow definitions
- Key files: `deploy.yml` (deploys to GitHub Pages on push to main)

**`.planning/codebase/`:**
- Purpose: GSD framework codebase analysis documents
- Contains: Architecture, structure, conventions, testing, stack analysis
- Key files: Generated by `/gsd:map-codebase` command

**`.claude/`:**
- Purpose: Claude AI configuration and GSD framework
- Contains: Agent definitions, commands, hooks, templates
- Key files: `settings.json`, agent markdown files, workflow definitions

## Key File Locations

**Entry Points:**
- `public/index.html`: Browser entry, HTML shell with `<div id="root">`
- `src/index.js`: JavaScript entry, bootstraps React app
- `src/App.js`: Application root component

**Configuration:**
- `package.json`: Dependencies, scripts, build config
- `.gitignore`: Git exclusions
- `src/config/supabase.js`: Supabase client (reads `REACT_APP_SUPABASE_URL` and `REACT_APP_SUPABASE_ANON_KEY`)

**Core Logic:**
- `src/App.js`: Main application component (currently placeholder)

**Styling:**
- `src/index.css`: Global styles (font families, body defaults)
- `src/App.css`: App component styles (header, animations)

**Testing:**
- Not yet implemented (React Testing Library installed but no test files)

## Naming Conventions

**Files:**
- Components: `PascalCase.js` (e.g., `App.js`)
- Entry files: `lowercase.js` (e.g., `index.js`)
- Stylesheets: Match component name with `.css` extension (e.g., `App.css`)
- Config files: `lowercase.js` (e.g., `supabase.js`)

**Directories:**
- Lowercase with hyphens for multi-word (e.g., `israel-housing-finder`)
- Lowercase single-word for subdirectories (e.g., `config`, `workflows`)

## Where to Add New Code

**New Feature Component:**
- Primary code: `src/components/<FeatureName>.js` (directory not yet created)
- Styles: `src/components/<FeatureName>.css`
- Tests: `src/components/<FeatureName>.test.js`

**New Page/Route:**
- Implementation: `src/pages/<PageName>.js` (directory not yet created)
- Wire into: `src/App.js` (add routing when implemented)

**Utilities:**
- Shared helpers: `src/utils/<utilName>.js` (directory not yet created)

**Services/API Integration:**
- Implementation: `src/services/<serviceName>.js` (directory not yet created)
- Configuration: `src/config/<serviceName>.js` (follows existing pattern)

**New External Service:**
- Client setup: `src/config/<service>.js`
- Environment vars: Add to `.env.local` (not committed, add to `.gitignore`)
- Documentation: Update DEVELOPMENT.md with required env vars

## Special Directories

**`node_modules/`:**
- Purpose: NPM installed dependencies
- Generated: Yes (via `npm install`)
- Committed: No (excluded in `.gitignore`)

**`build/`:**
- Purpose: Production build output
- Generated: Yes (via `npm run build`)
- Committed: No (excluded in `.gitignore`, deployed via CI/CD)

**`.planning/`:**
- Purpose: GSD framework planning documents
- Generated: Partially (some files generated by GSD commands)
- Committed: Yes (planning artifacts tracked in git)

**`.claude/`:**
- Purpose: Claude AI configuration, GSD framework tooling
- Generated: No (part of GSD framework setup)
- Committed: Currently untracked (shown in git status as `?? .claude/`)
